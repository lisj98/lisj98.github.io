​ Spring MVC 通过一套 MVC 注解，让 POJO 成为处理请求的控制器，而不用像原生 Spring 那样需要事先实现接口。

​ Spring MVC 的核心是 DispatcherServlet ，DispatcherServlet 是Spring MVC的总导演、总策划，是Spring MVC的灵魂和心脏，它负责截获客户端发来的请求并协调好 Spring MVC 的各个组件来完成请求处理并返回响应的过程。

1.1 体系结构

​ Spring MVC，顾名思义，是 MVC(Model、View、Control) 的集中体现，它利用处理器来分离模型、视图和控制，达到不同层级间松散耦合的效果，是基于 Model 2 实现的技术框架，而 Model 1 就是把所有的展现逻辑和业务逻辑集中到一起，缺点自然就是紧耦合、复用性差、维护成本高。

Spring MVC 框架模型：

​ 从 Spring MVC 框架模型中可以看出，DispatcherServlet 处于核心的地位，它负责协调和组织不同组件以完成请求处理并返回相应的工作。Spring MVC 通过一个前端 Servlet 接收所有的请求，并将具体工作委派给其他组件进行处理，DispatcherServlet 就是 Spring MVC 的前端 Servlet。

Spring MVC处理请求的整体过程 ：

  1. 客户端发出一个 HTTP 请求，Web 应用服务器接收到这个请求，如果匹配到了 DispatcherServlet 指定的请求映射路径（在 web.xml 中指定），则将该请求转交给 DispatcherServlet 处理。
  2. DispatcherServlet 接收到这个请求后，根据请求的信息（URL、HTTP 请求方法、请求报文头、请求参数、Cookie 等）及 HandlerMapping 的配置找到处理请求的特定处理器（Handler）。
  3. 当 DispatcherServlet 得到具体的处理器后，通过 HandlerAdapter 对 Handler 进行封装，再以统一的适配器接口调用 Handler。HandlerAdapter 是一个适配器，它用统一的接口对各种 Handler 方法进行调用。
  4. 当处理器（Handler）完成业务逻辑的处理后将返回一个 ModelAndView 给 DispatcherServlet ，ModelAndView 包含了逻辑视图名和模型数据信息。
  5. ModelAndView 中包含的“逻辑视图名”并不是真正的视图对象，需要通过 ViewResolver 完成逻辑视图名到真实视图对象的解析工作。
  6. 当得到真实的视图对象 View 后，DispatcherServlet 就用这个 View 对象对 ModelAndView 中的模型数据进行视图渲染。
  7. 最终客户端得到的就是一个真实的视图，可以是一个 HTML 页面，也可以是一个 XML 串、一个 JSON 串、一张图片等。

1.2 配置 DispatcherServlet

​ 要了解 Spring MVC 框架的工作原理，必须回答三个问题：

  1. DispatcherServlet 如何截获特定的 HTTP 请求并交由框架进行处理？（通过 <servlet-mapping>）
  2. 位于 Web 层的 Spring 容器（WebApplicationContext）如何与位于业务层的 Spring 容器（ApplicationContext）建立联系，以使 Web 层的 Bean 可以调用业务层的 Bean？（一个 web.xml 文件可以指定多个 DispatcherServlet，通过 <servlet-mapping> 配置，让每个 DispatcherServlet 处理不同的请求）
  3. 如何初始化 Spring MVC 的各个组件，并将它们装配到 DispatcherServlet 中？（initStrategies() 方法）
配置 DispatcherServlet ，截获特定的 HTTP 请求

传统的 web.xml 的方式：

​ 一个 web.xml 可以配置多个 DispatcherServlet ，通过其 <servlet-mapping> 配置，让每个 DispatcherServlet 处理不同的请求。

​ DispatcherServlet 遵循 约定大于配置 的原则，在大多数情况下，用户无须进行额外的配置，只需要按照约定行事即可。

​ 如果确实要对 DispatcherServlet 的默认规则进行调整的话，可以通过<servlet>的<init-param>指定：

  • namespace：DispatcherServlet 对应的命名空间，默认为<servlet-name>-servlet，用于构造 Spring 配置文件的路径。在显式指定路径后，配置文件对应的路径为 /WEB-INF/<namespace>.xml，而不是 /WEB-INF/<servlet-name>-servlet.xml。例如上述代码中的配置文件就为 smart-servlet.xml，而要是 namespace 指定为 smart，则配置文件就为 smart.xml。
  • contextConfigLocation：用于指定业务层或持久层的配置文件，即 <param-value> 中的文件。
  • publishContext：布尔类型，默认为 true。DispatcherServlet 根据该属性决定是否将 WebApplicationContext 发布到 ServletContext 的属性列表中，以便调用者可借由 ServletContext 找到 WebApplicationContext 实例，对应的属性名为 DispatcherServlet#getContextAttributeName() 方法的返回值。
  • publishEvents：布尔类型。当 DispatcherServlet 处理完一个请求后，是否需要向容器发布一个 ServletRequestHandledEvents 事件，默认为true。

编程的方式（Servlet 3.0的实现方式）：

​ 也就是把 web.xml 中的实现方式用 Java 代码实现，仔细比较的话其实是一样的。

​ Servlet 3.0 的实现原理：在 Servlet 3.0 的环境中，容器会在类路径中查找实现 javax.servlet.ServletContainerInitializer 的类，如果发现已有实现类，就会调用它来配置 Servlet 容器。在 Spring 中，org.springframework.web.SpringServletContainerInitializer 类实现了该接口，同时这个类又会查找实现 org.springframework.web.WebApplicationInitializer 接口的类，并将配置任务交给这些实现类去完成。另外，Spring 提供了一个便利的抽象类 AbstractAnnotationConfigDispatcherServletInitializer 来实现这个接口，使得它在注册 DispatcherServlet 时只需简单地指定它的 Servlet 映射即可。在上述示例中，当应用部署到 Servlet 3.0 容器中时，容器启动时会自动发现它，并使用它来配置 Servlet 上下文。

探究 DispatcherServlet 的内部逻辑

​ 通过查看 DispatcherServlet 的 initStrategies() 方法的代码，就能知道 Spring 是如何将上下文中的 Spring MVC 组件装配到 DispatcherServlet 中的了。

​ initStrategies() 方法会在 WebApplicationContext 初始化后自动执行，此时 Spring 上下文中的 Bean 已经初始化完毕。该方法的工作原理是：通过反射机制查找并装配 Spring 容器中用户显式自定义的组件 Bean，如果找不到，则装配默认的组件实例。

​ DispatcherServlet 装配各型组件的逻辑：

​ 有些组件最多允许存在一个实例，用☆表示；而另一些组件允许存在多个实例，用★表示。这些组件都实现了 org.springframework.core.Ordered 接口，同一类型的组件如果存在多个，可以通过 order 属性确定优先级顺序，值越小优先级越高。

​ 简要描述一下 Spring MVC 处理 /user.html 的整个过程：

  1. DispatcherServlet 接收到客户端的 /user.html 请求。
  2. DispatcherServlet 使用 DefaultAnnotationHandlerMapping 查找负责处理该请求的处理器。
  3. DispatcherServlet 将请求分发给名为 /user.html 的 UserController处理器。
  4. 处理器完成业务处理后，返回 ModelAndView 对象，其中 View 的逻辑视图名为 /user/createSuccess，而模型包含一个 user 的模型数据信息。
  5. DispatcherServlet 调用 InternalResourceViewResolver 组件对 ModelAndView 中的逻辑视图名进行解析，得到真实的 /WEB-INF/view/user/createSuccess.jsp 视图对象。
  6. DispatcherServlet 使用/WEB-INF/view/user/createSuccess.jsp 对模型中的 user 模型对象进行渲染。
  7. 返回相应页面给客户端。
